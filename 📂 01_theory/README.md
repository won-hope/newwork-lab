## 웹브라우저 주소를 치면 일어나는 과정

1) DNS Resoultion
### 1. 브라우저 캐시 (Browser Cache)
가장 먼저 웹 브라우저 자체의 내장 메모리를 확인하는 단계입니다.
* **동작:** 크롬, 사파리 등 브라우저 내부에 이전에 해당 웹 사이트를 방문하여 저장된 도메인과 IP 주소의 매핑 기록이 남아있는지 탐색합니다.

### 2. OS 캐시 (OS Cache)
브라우저 캐시에 정보가 없다면, 시스템 콜(System Call)을 통해 운영체제(OS) 단으로 넘어갑니다.
* **동작:** OS가 자체적으로 관리하는 로컬 DNS 캐시 메모리나 `hosts` 파일 내에 해당 도메인에 매핑된 IP 주소가 있는지 탐색합니다.

### 3. 라우터 캐시 (Router Cache)
내부 PC(OS)에서도 IP를 찾지 못하면, 로컬 네트워크를 타고 다음 관문으로 이동합니다.
* **동작:** PC와 랜선/와이파이로 연결된 기본 게이트웨이(가정 및 사무실의 라우터/공유기)로 쿼리를 전송하여 라우터 내부의 자체 DNS 캐시를 확인합니다.

### 4. ISP DNS 서버 (ISP DNS / Recursive DNS)
내부 네트워크를 완전히 벗어나 외부 인터넷 망으로 나가는 본격적인 단계입니다.
* **동작:** 가입된 인터넷 통신사(ISP - KT, SKT 등)의 DNS 서버에 해당 도메인의 IP 정보를 요청합니다.
* **💡 심화 (Recursive Query):** 만약 ISP DNS 서버 캐시에도 해당 IP가 없다면, ISP DNS 서버가 직접 **루트(Root) DNS ➡️ 최상위(TLD) DNS ➡️ 권한(Authoritative) DNS** 순으로 전 세계의 네임 서버를 순회하며 최종 IP를 알아냅니다. 이후 결과를 클라이언트(PC)에게 반환하고 자신의 캐시에 저장해 둡니다.
